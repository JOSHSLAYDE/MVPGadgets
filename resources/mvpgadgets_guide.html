MVPGadgets Usage Guide

This is a reference to commands and permissions which you as an administator should be aware of.

Commands

  • /mvpgadgets reload

    • Reloads all MVPGadgets config files.
    • Aliases: mvp, mvpg
    • Permission: mvpgadgets.reload

  • /gadget

    • Displays Gadget Selector GUI, sets item in hand to GUI Gadget.
    • Permission: mvpgadgets.gadget

  • /mobcannon

    • Help menu for the MobCannon commands.
    • Aliases: mbc
    • Permission: mvpgadgets.mobcannon.use

  • /mobcannonreload

    • Reloads the MVPGadgets main config file, which contains MobCannon settings.
    • Aliases: mobcannonrl, mcreload, mcrl, reloadmobcannon
    • Permission: mvpgadgets.mobcannon.reload

  • /launchmob

    • If player does not specify a mob, this will launch a random mob.
    • Aliases: launch, lmob, lm
    • Permission: mvpgadgets.mobcannon.launchmob.random

  • /launchmob <mob name>

    • Launches a specific mob.
    • Aliases: mobcannonrl, mcreload, mcrl, reloadmobcannon
    • Permission: mvpgadgets.mobcannon.launchmob

  • /moblist

    • Lists all the mobs that the player has permission to launch.
    • Aliases: listmobs, launcherlist, launchlist
    • Permission: mvpgadgets.mobcannon.moblist

  • /mobnames <mob name>

    • List all alternate names for a given mob name, according to the MVPGadgets config. (i.e. ocelot and cat)
    • Aliases: altmobnames, altmobs
    • Permission: mvpgadgets.mobcannon.mobnames

Permissions

  • mvpgadgets.reload

    • Use /mvpgadgets reload

  • mvpgadgets.gadget

    • Use /gadget

    • mvpgadgets.<gadgetName>

    • Grants the usage of a particular gadget based on its name in the config (or messages) file.
      • The gadget name should be in a programming-variable format, with the first letter lowercased. Ex: mvpgadgets.horseGadget
      • This is case-sensitive. Again, look in the config (or messages) file for exact names.

This last bunch is all MobCannon command permissions (not to be confused with the MobCannon Gadget).

  • mvpgadgets.mobcannon.use

    • Base help command for MobCannon.
    • Parents: all other mvpgadgets.mobcannon.<node> permissions
    • Children: none

  • mvpgadgets.mobcannon.launchmob

    • Base permission to be able to launch a mob.
    • Parents:
      • mvpgadgets.mobcannon.launchmob.random
      • mvpgadgets.mobcannon.launchmob.all
      • mvpgadgets.mobcannon.launchmob.*
      • mvpgadgets.mobcannon.*
    • Children: none

  • mvpgadgets.mobcannon.launchmob.random

    • Launch random mobs without specification in the /launchmob [mob] command.
    • Parents:
      • mvpgadgets.mobcannon.launchmob.*
      • mvpgadgets.mobcannon.*
    • Children:
      • mvpgadgets.launchmob

  • mvpgadgets.mobcannon.launchmob.all

    • Permission to all mobs, but does not grant access to the random launcher.
    • Parents:
      • mvpgadgets.mobcannon.launchmob.*
      • mvpgadgets.mobcannon.*
    • Children:
      • mvpgadgets.launchmob

  • mvpgadgets.mobcannon.launchmob.*

    • Permission to all mobs, as well as the random launcher.
    • Parents:
      • mvpgadgets.mobcannon.*
    • Children:
      • mvpgadgets.launchmob
      • mvpgadgets.launchmob.random
      • mvpgadgets.launchmob.all

  • mvpgadgets.mobcannon.moblist

    • List all available mobs, meaning those that the user has permission for and, ultimately, that the config file allows.
    • Parents:
      • mvpgadgets.mobcannon.*
    • Children: none

  • mvpgadgets.mobcannon.mobnames

    • List all alternate names for a given mob name.
    • Parents:
      • mvpgadgets.mobcannon.*
    • Children: none

  • mvpgadgets.mobcannon.reload

    • Reload MobCannon.
    • Parents:
      • mvpgadgets.mobcannon.*
    • Children: none

  • mvpgadgets.mobcannon.*

    • Grants all permissions related to MobCannon (aside from the usage of the MobCannon Gadget).
    • Parents:
      • mvpgadgets.mobcannon.*
    • Children: none
